\FOIL{Computing the transitive closure}
\vfill
Can be described as a new state-space exploration problem:
\LIST{
\BULL Potential state space: $(\vi, \vj)$ where $\vi, \vj \in \sset_{rch}$.
\BULL Initial states: $\{(\vi, \vj)|(\vi, \vj) \in \nset^{-1}\}$.
\BULL Next-state function $\nset'$:
$$
\nset'(~(\vi, \vj)~) \!=\! \{(\vi, \vk)|\vk \in \nset^{-1}(\vj)\}
$$
}

Our algorithm using saturation is based on the following observation:
$$
\mbox{if} ~ (\vi, \vk) \in \nset^{-1} ~ \mbox{then} ~ (\vi, \vj) \in TC^{-1}
\mbox{ ~ where ~}
\vj \in \id{Saturate}(\{\nset^{-1}_{L}, \cdots, \nset^{-1}_{1}\},\{\vk\})
$$

Top-level pseudocode:

\ALGORITHM{$mdd ~~ \id{SCC\_TC}(\nset^{-1})$} {
\LISTING{0}{
%\LL $mdd~N \!\ASSIGN\!   \id{QQencode}(\nset^{-1})$;
\LL $mdd~TC^{-1} \!\ASSIGN\!  \id{TransClosureSat}(\nset^{-1})$; \label{line:TC}
\LL $mdd~SCC \!\ASSIGN\! \id{TCtoSCC}(TC^{-1})$; \REMARK{Finding all $(\vi,\vi) \in TC^{-1}$}
\LL return $SCC$;
}}
